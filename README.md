# Light Image Compressor

一个轻量、无第三方依赖的 Windows 图片压缩工具。  
目标很直接：输入一张图片，指定目标大小（KB），尽量保住画质，把输出控制在接近目标大小的范围内。

这个工具目前是一个单文件原生 Win32 GUI 程序，核心代码在 [`transform.cpp`](./transform.cpp)。

## 特性

- 指定目标大小（KB）压缩图片
- 图形化界面，支持按钮选图和拖拽图片
- 优先通过 JPEG 质量调节压缩，尽量保留观感
- 如果单纯降低 JPEG 质量仍达不到目标，会自动适度缩小分辨率
- 自动根据 Windows UI 语言切换中文 / 英文界面
- 单文件原生实现，使用 Win32 + WIC，无 Qt / OpenCV / 其他大型依赖
- 输出为 JPEG，体积小，适合做轻量桌面小工具

## 适用场景

- 想把图片压到 500KB、300KB、100KB 这类明确大小
- 不想装大型图片软件，只需要一个简单 GUI 工具
- 想快速处理截图、照片、上传素材
- 希望工具本身体积尽量小、依赖尽量少

## 当前行为

- 输入格式支持：`jpg`、`jpeg`、`png`、`bmp`、`gif`、`tif`、`tiff`
- 输出格式固定为 `JPEG`
- 如果原图有透明通道，输出 JPEG 时会自动铺白底
- 如果目标大小过小，程序可能提示无法达到目标
- 输出文件扩展名会自动整理为 `.jpg`

## 压缩策略

为了尽量保住图片质量，程序不是直接粗暴缩图，而是按下面的顺序尝试：

1. 优先保持原始分辨率
2. 对 JPEG 质量做二分搜索，寻找能满足目标大小的最高质量
3. 如果最低质量仍然太大，再逐步缩小分辨率
4. 在多个可行结果里，优先选择画质更高、缩放更少的方案

这意味着：

- 当目标大小比较宽松时，通常只会降低 JPEG 质量，不会缩图
- 当目标大小很紧时，程序才会自动降低分辨率

## 界面使用

1. 打开程序
2. 选择图片，或者把图片直接拖进窗口
3. 输入目标大小（单位 `KB`）
4. 确认输出路径
5. 点击 `Compress / 开始压缩`
6. 在下方结果区域查看压缩结果、输出大小、最终质量和分辨率

## 构建

当前已验证的构建方式是 `MinGW-w64 g++`：

```bash
g++ transform.cpp -std=c++17 -municode -mwindows -Wall -Wextra -lole32 -loleaut32 -lwindowscodecs -lcomdlg32 -lshell32 -o image_compressor.exe
```

构建完成后会生成：

```text
image_compressor.exe
```

## 技术实现

- GUI：Win32 API
- 图片解码 / 缩放 / JPEG 编码：Windows Imaging Component（WIC）
- 并发：后台线程执行压缩，避免界面卡死
- 本地化：根据 Windows UI 语言自动切换中英文

## 为什么它比较轻

这个项目没有引入 Qt、Electron、OpenCV、ImageMagick 之类的大型依赖，直接使用 Windows 自带 API 完成：

- 文件选择
- 拖拽接收
- 图片解码
- 图片缩放
- JPEG 编码

因此代码结构简单，编译产物也比较小，适合当作随手可用的小工具。
